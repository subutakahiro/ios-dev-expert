# Test code Generation Rules

- アプリケーションの生成と合わせ、ビジネスロジックとユーザーインターフェースの単体テストコードを生成する。
- アプリケーションに機能の追加を行った場合、ビジネスロジックとユーザーインターフェースの単体テストコードを生成する。
- アプリケーションの機能を削除した場合、対応するビジネスロジックとユーザーインターフェースの単体テストコードを削除する。
- アプリケーションの機能の追加や削除を行った場合、未使用のデッドコードができた時は該当するソースコードを削除する。
- XCTestではなく、最新のSwift Testingフレームワークを使用したテストコードを生成する。
